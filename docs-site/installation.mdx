Manifest is published as `@angriff36/manifest` to GitHub Packages. Before installing, make sure your project meets the minimum requirements: **Node.js 18 or later** and a package manager that supports scoped registries (npm or pnpm).

<Note>
  **Current version:** `0.6.1`. See [Migration](/migration/vnext-migration-guide) for what changed since 0.5.x.
</Note>

## Install the package

<CodeGroup>
  ```bash npm theme={null}
  npm install @angriff36/manifest
  ```

  ```bash pnpm theme={null}
  pnpm add @angriff36/manifest
  ```
</CodeGroup>

<Warning>
  `@angriff36/manifest` is published to GitHub Packages, not the public npm registry. You need a GitHub personal access token with `read:packages` scope and an `.npmrc` entry that routes `@angriff36` to `https://npm.pkg.github.com`. See GitHub's [Authenticating to GitHub Packages](https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-npm-registry#authenticating-to-github-packages) documentation for setup instructions.
</Warning>

## Package exports

The package ships multiple named entry points. Import only what you need â€” each export is independent.

| Import path                              | What it provides                                    |
| ---------------------------------------- | --------------------------------------------------- |
| `@angriff36/manifest`                    | `RuntimeEngine` â€” the main command execution engine |
| `@angriff36/manifest/compiler`           | The high-level `compiler` orchestrator              |
| `@angriff36/manifest/ir-compiler`        | `compileToIR` â€” parse source and produce IR v1      |
| `@angriff36/manifest/ir`                 | IR data structure types                             |
| `@angriff36/manifest/stores`             | Node.js store adapters                              |
| `@angriff36/manifest/projections/nextjs` | Next.js code generation projection                  |
| `@angriff36/manifest/projections/routes` | Route generation projection                         |
| `@angriff36/manifest/registry/emit`      | Event registry emit helpers                         |
| `@angriff36/manifest/audit`              | `AuditSink` interface                               |
| `@angriff36/manifest/audit/memory`       | `MemoryAuditSink` for development and tests         |
| `@angriff36/manifest/audit/postgres`     | `PostgresAuditSink` for production                  |
| `@angriff36/manifest/outbox`             | `OutboxStore` interface                             |
| `@angriff36/manifest/outbox/memory`      | `MemoryOutboxStore` for development and tests       |
| `@angriff36/manifest/outbox/postgres`    | `PostgresOutboxStore` for production                |

The most common imports for getting started are:

```typescript theme={null}
import { RuntimeEngine } from '@angriff36/manifest';
import { compileToIR } from '@angriff36/manifest/ir-compiler';
```

## TypeScript setup

The package ships declaration files for every export. No `@types/` package is needed. Your `tsconfig.json` must target **ES2020 or later** and enable `moduleResolution: "bundler"` or `"node16"`/`"nodenext"` to resolve the package's `exports` map correctly.

```json theme={null}
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true
  }
}
```

<Note>
  The package is published as ESM (`"type": "module"`). If your project uses CommonJS, configure your bundler or TypeScript to handle ESM interop, or use a bundler like Vite or esbuild that handles this automatically.
</Note>

## Optional peer dependency: Supabase

The Supabase adapter is an optional peer dependency. Install it only if you plan to use Supabase as a backing store:

```bash theme={null}
npm install @supabase/supabase-js
```

The package declares `@supabase/supabase-js@^2` as an optional peer. If it is not installed, all non-Supabase store targets continue to work normally.

## Install the CLI

The `manifest` CLI ships inside the package under `packages/cli/dist/index.js` and is registered as the `manifest` binary. It is available immediately after installing `@angriff36/manifest`:

```bash theme={null}
# Verify the CLI is available
npx manifest --help
```

The CLI provides commands grouped into three areas:

```bash theme={null}
# Build pipeline (init, compile, generate, build, validate)
manifest init
manifest compile modules/recipe.manifest --output ir/
manifest generate ir/recipe.ir.json --projection nextjs --output app/api/
manifest build modules/recipe.manifest

# Governance and audit (v0.6.0+)
manifest enforce-surface          # detect duplicate/bypass write paths
manifest integration-check        # umbrella validator for downstream consumers
manifest audit-governance         # 5-detector governance suite
manifest audit-bypasses           # validate the bypass registry

# Diagnostics
manifest doctor
manifest scan
```

See the [CLI overview](/cli/overview) for the full command list.

<Tip>
  Run `manifest init` once at the start of a project. It creates `manifest.config.yaml` with your framework, auth provider, import paths, and output directory. After that, `manifest build` reads those defaults and you only need to pass flags when overriding them.
</Tip>

## Verifying your install

Run the following to confirm the package resolves correctly and the CLI responds:

```bash theme={null}
# Check CLI
npx manifest --help

# Smoke-test the compiler from a script
node --input-type=module <<'EOF'
import { compileToIR } from '@angriff36/manifest/ir-compiler';
const { ir, diagnostics } = await compileToIR('entity Ping { property id: string }');
console.log('IR version:', ir?.version ?? 'compiled OK');
EOF
```

## Windows note

On Windows with pnpm, the `manifest` shim may exit with code `0` and no output on older versions. If this happens, invoke the CLI directly:

```bash theme={null}
pnpm exec manifest --help
# or
node .\node_modules\@manifest\runtime\packages\cli\dist\index.js --help
```

This was fixed in `0.3.10` â€” updating to the latest version resolves it.
