The `timestamps` entity modifier auto-injects read-only `createdAt` and `updatedAt` properties and populates them at runtime, removing the boilerplate of declaring and maintaining audit timestamps by hand.

Matches `docs/spec/semantics.md` / IR `timestamps` flag / runtime `getNow()` create+update stamping / Prisma `@default(now())` + `@updatedAt`. Package pin `@angriff36/manifest` **3.6.41**.

## Syntax

The modifier is a bare keyword placed inside an entity body. No property declarations for `createdAt` or `updatedAt` are needed:

```manifest
entity Article {
  property required id: string
  property required title: string
  property body: string = ""
  timestamps
}

store Article in memory
```

## Behavior

When an entity declares `timestamps`, the IR compiler sets `IREntity.timestamps: true` and injects two properties — `createdAt` and `updatedAt`, both of type `datetime` with the `readonly` modifier — unless the entity already declares a property of that name. The injection is idempotent and skips names that are already present.

The runtime engine acts on the flag:

- On instance **creation**, when `timestamps` is set, both `createdAt` and `updatedAt` are assigned the value of `getNow()`.
- On instance **update**, when `timestamps` is set, `updatedAt` is reassigned to `getNow()` while `createdAt` is left untouched.

`getNow()` honors the deterministic clock supplied through the `now` runtime option, so tests can pin timestamps to a fixed value. The Prisma projection translates the flag into native column attributes: `@default(now())` on `createdAt` and `@updatedAt` on `updatedAt`.

## Manual `now()` / `today()` defaults

The `timestamps` modifier is the zero-boilerplate path for the `createdAt`/`updatedAt` pair. For a single timestamp field outside that pair, declare it explicitly with a `now()` (or `today()`) default:

```manifest
property createdAt: datetime = now()
```

This call-expression default lowers to the `IRProperty.autoNow` flag (recognized: `now()`/`today()`, called with no arguments). The runtime stamps the field with the current time on create when the caller does not supply a value, and the Prisma projection emits `@default(now())`.

<Note>
  Before this lowering existed, a `= now()` default compiled to **no** default and was null-filled
  at runtime, breaking non-null store columns (the `createdAt must not be null` class of bug). The
  compiler also folds negative numeric literal defaults like `= -1` into a real static default, and
  emits a **warning** for an unsupported call-expression default such as `= uuid()` (which would
  otherwise be silently dropped). See [Types](/language/types) for the full `autoNow` and
  diagnostics behavior.
</Note>

## Reference

- Source keyword: `timestamps`.
- IR field: `IREntity.timestamps?: boolean`.
- Injected properties: `createdAt`, `updatedAt`, type `datetime`, modifier `readonly`.
- Runtime population: `getNow()` on create (both fields) and update (`updatedAt` only).
- Prisma attributes: `@default(now())`, `@updatedAt`.

<Note>
  The injected fields carry the `readonly` modifier, so they cannot be mutated through commands. If
  a program manually declares `createdAt` or `updatedAt`, the auto-injection defers entirely to the
  manual declaration and does not add or override anything for that name. Population is driven by
  the runtime engine's create/update paths — stores or code paths that write instances without going
  through `createInstance`/`updateInstance` will not receive automatic timestamps.
</Note>
