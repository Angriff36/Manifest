Builder is a browser-based studio and application-generation control plane for [Manifest](https://manifest.build). You edit one or more `.manifest` source files, compile them to Manifest IR, inspect the resulting model, generate projection artifacts, and assemble runnable application packages.

Builder does not replace Manifest. Manifest owns the language, compiler, IR, runtime semantics, and projection implementations. Builder owns the authoring environment, project orchestration, visualization, assembly presets, and generated-application lifecycle.

## The core workflow

```text
.manifest source
      |
      v
Manifest compiler -> IR + diagnostics
      |                    |
      |                    +-> graphs, checks, flows, metrics, symbols
      v
Manifest projections -> generated source artifacts
      |
      v
Builder assembly -> project file map + assembly report
      |
      v
initial build or ownership-aware update
```

<Info>
  Builder can run locally without a backend. Supabase configuration is optional and enables sign-in,
  cloud project synchronization, share links, and the authenticated AI edge function.
</Info>

## Safety boundary

Assembly creates a desired file map. Filesystem project generation is a separate operation with two explicit modes:

- **Initial** creates a new application and records every generated file in `.builder/ownership.json`.
- **Update** requires that ownership manifest, previews the plan, and changes only recognized Builder-owned files.

Builder never implicitly deploys Convex, initializes a deployment, changes environment variables, runs migrations or seeds, or clears application data.
