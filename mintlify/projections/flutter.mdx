The Flutter projection generates type-safe Dart model classes from compiled Manifest IR. Use it when you are building a Flutter or Dart application and want Manifest entities represented as first-class Dart types with JSON serialization, equality, `copyWith`, constraint validation, and optional state management integration via Riverpod or Provider. The projection follows the same compile-time, app-agnostic contract as the [Pydantic projection](/projections/python-pydantic) but targets Dart conventions.

The projection is registered under the name `dart` and lives at `src/manifest/projections/dart/generator.ts`.

## What it generates

The projection exposes six surfaces.

`dart.entity` emits a single Dart model class for one entity. The class includes `fromJson`/`toJson` factory methods, a `copyWith` method for immutable updates, `==`/`hashCode` overrides for value equality, and a `validate()` method that checks Manifest constraints against property values.

`dart.command` emits a Dart class for command parameters and an optional return type class. Both include JSON serialization support.

`dart.models` emits all entity and command models in a single library file with barrel exports.

`dart.client` emits a Dio-based async HTTP client class with CRUD methods (`list`, `get`, `delete`) for each entity and typed command invocation methods. The client uses `httpx.AsyncClient`-equivalent Dio patterns with optional bearer token authentication.

`dart.providers` emits state management hooks supporting Riverpod (`Provider`, `FutureProvider`, `FutureProvider.family`) or classic Provider (`ChangeNotifier`). Choose the pattern via the `stateManagement` option.

`dart.package` emits a complete Flutter package: models, client, providers, a `pubspec.yaml`, and a `README.md`.

## Usage

```ts
import { getProjection } from '@angriff36/manifest/projections';

const projection = getProjection('dart');

// Generate a single entity model
const entity = projection.generate(ir, {
  surface: 'dart.entity',
  entity: 'Order',
});

// Generate all models in one file
const models = projection.generate(ir, { surface: 'dart.models' });

// Generate a complete Flutter package
const pkg = projection.generate(ir, {
  surface: 'dart.package',
  options: {
    stateManagement: 'riverpod',
    clientBaseUrl: 'https://api.example.com',
    packageName: 'manifest_models',
  },
});
```

Through the CLI:

```bash
manifest generate ir/widget.ir.json --projection dart --surface dart.models --output lib/models/manifest_models.dart
```

## Type mapping

The projection maps Manifest IR types to Dart types:

| Manifest type      | Dart type        |
| ------------------ | ---------------- |
| `string`           | `String`         |
| `int`              | `int`            |
| `number` / `float` | `double`         |
| `boolean` / `bool` | `bool`           |
| `datetime`         | `DateTime`       |
| `uuid`             | `String`         |
| `decimal`          | `String`         |
| `array<T>`         | `List<T>`        |
| `map<V>`           | `Map<String, V>` |

Nullable types use Dart's `T?` syntax with null-checked JSON parsing. Enum types generate a Dart `enum` class with a `fromString` lookup factory. Computed properties are emitted as getter stubs annotated with a comment indicating they must be evaluated server-side.

## Constraint validation

Manifest constraints are mapped to `validate()` methods on generated entity classes. The projection uses the shared `constraint-analysis.ts` utilities:

- Numeric range constraints (`min`, `max`) become `if (field < min || field > max)` checks.
- String length constraints (`minLength`, `maxLength`) become `if (field.length < min || field.length > max)` checks.
- Pattern / regex constraints become `if (!RegExp(pattern).hasMatch(field))` checks.

The `validate()` method returns a list of violation descriptions, or an empty list when all constraints pass.

## Options

`DartProjectionOptions` accepts `stateManagement` (`'riverpod'` | `'provider'` | `'none'`, default `'riverpod'`), `clientBaseUrl` (default `'http://localhost:3000'`), `clientClassName` (default `'ManifestClient'`), `packageName` (default `'manifest_models'`), `emitPackageFiles` (default `false`), and `customImports` (additional import statements to include in generated files).

## Generated code example

For an entity `Order` with properties `id`, `total`, and `status`:

```dart
class Order {
  final String id;
  final double total;
  final String status;

  const Order({
    required this.id,
    required this.total,
    required this.status,
  });

  factory Order.fromJson(Map<String, dynamic> json) => Order(
    id: json['id'] as String,
    total: (json['total'] as num).toDouble(),
    status: json['status'] as String,
  });

  Map<String, dynamic> toJson() => {
    'id': id,
    'total': total,
    'status': status,
  };

  Order copyWith({String? id, double? total, String? status}) => Order(
    id: id ?? this.id,
    total: total ?? this.total,
    status: status ?? this.status,
  );

  List<String> validate() {
    final violations = <String>[];
    if (total < 0) violations.add('total must be >= 0');
    return violations;
  }

  @override
  bool operator ==(Object other) =>
    identical(this, other) ||
    other is Order && id == other.id && total == other.total && status == other.status;

  @override
  int get hashCode => Object.hash(id, total, status);
}
```

## Notes & limitations

<Note>
  The projection generates syntactically valid Dart code with `camelCase` field names and
  `PascalCase` class names. The Dio HTTP client is the default networking layer for the
  `dart.client` surface. Computed properties are emitted as getter stubs only — they carry a comment
  noting that the value must be computed server-side, since Manifest computed properties are
  evaluated by the runtime engine, not by the client.
</Note>

The `dart.providers` surface generates Riverpod or Provider state management hooks that wrap the HTTP client methods. The generated providers handle loading, error, and data states following the conventions of the chosen state management library.
