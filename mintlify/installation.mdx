Manifest is published as `@angriff36/manifest` to the public npm registry. Before installing, make sure your project meets the minimum requirements: **Node.js 18 or later** and npm or pnpm.

<Note>
  **Current version:** `2.18.0`. The package is fully self-contained: the compiler, runtime, CLI, MCP server, and LSP server all ship inside `@angriff36/manifest`, and their runtime dependencies are declared and installed automatically — no manual dependency installs are needed.
</Note>

## Install the package

<CodeGroup>

```bash npm
npm install @angriff36/manifest
```

```bash pnpm
pnpm add @angriff36/manifest
```

</CodeGroup>

<Note>
  `@angriff36/manifest` resolves from the public npm registry — no GitHub Packages authentication, personal access token, or custom `.npmrc` registry entry is required. `npm install` / `pnpm add` works out of the box.
</Note>

## Package exports

The package ships multiple named entry points. Import only what you need — each export is independent.

| Import path | What it provides |
|-------------|-----------------|
| `@angriff36/manifest` | `RuntimeEngine` — the main command execution engine |
| `@angriff36/manifest/runtime-engine` | Full runtime module: `RuntimeEngine`, stores, event types |
| `@angriff36/manifest/binary-ir` | Binary IR pack/unpack (MessagePack encoding) |
| `@angriff36/manifest/compiler` | The high-level `compiler` orchestrator |
| `@angriff36/manifest/ir-compiler` | `compileToIR` — parse source and produce IR v1 |
| `@angriff36/manifest/ir` | IR data structure types |
| `@angriff36/manifest/stores` | Node.js store adapters |
| `@angriff36/manifest/projections/nextjs` | Next.js code generation projection |
| `@angriff36/manifest/projections/routes` | Route generation projection |
| `@angriff36/manifest/registry/emit` | Event registry emit helpers |
| `@angriff36/manifest/audit` | `AuditSink` interface |
| `@angriff36/manifest/audit/memory` | `MemoryAuditSink` for development and tests |
| `@angriff36/manifest/audit/postgres` | `PostgresAuditSink` for production |
| `@angriff36/manifest/outbox` | `OutboxStore` interface |
| `@angriff36/manifest/outbox/memory` | `MemoryOutboxStore` for development and tests |
| `@angriff36/manifest/outbox/postgres` | `PostgresOutboxStore` for production |

The most common imports for getting started are:

```typescript
import { RuntimeEngine } from '@angriff36/manifest';
import { compileToIR } from '@angriff36/manifest/ir-compiler';
```

## TypeScript setup

The package ships declaration files for every export. No `@types/` package is needed. Your `tsconfig.json` must target **ES2020 or later** and enable `moduleResolution: "bundler"` or `"node16"`/`"nodenext"` to resolve the package's `exports` map correctly.

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true
  }
}
```

<Note>
  The package is published as ESM (`"type": "module"`). If your project uses CommonJS, configure your bundler or TypeScript to handle ESM interop, or use a bundler like Vite or esbuild that handles this automatically.
</Note>

## Optional peer dependencies

Store and platform adapters that require external services are declared as **optional peer dependencies**. Install only the ones you use — everything else works without them:

| Package | Needed for |
|---------|-----------|
| `@supabase/supabase-js` | Supabase store target |
| `ioredis` | Redis outbox store |
| `mongodb` | MongoDB store adapter |
| `@libsql/client` | Turso store adapter |
| `@assemblyscript/loader` | WASM compute modules |

```bash
# Example: only if you use the Redis outbox
npm install ioredis
```

If an optional peer is not installed, the corresponding adapter fails with a clear error when used; all other store targets continue to work normally.

## Bundled binaries

Installing `@angriff36/manifest` registers three binaries, all working out of the box with no extra installs:

| Binary | What it is |
|--------|-----------|
| `manifest` | The CLI — compile, generate, validate, scan |
| `manifest-mcp` | Model Context Protocol server for AI agents |
| `manifest-lsp` | Language Server Protocol server for editor integration |

```bash
# Verify the CLI is available
npx manifest --help
```

The CLI provides four main commands:

```bash
# Interactive project setup — creates manifest.config.yaml
manifest init

# Compile .manifest source to IR
manifest compile modules/recipe.manifest --output ir/

# Generate code from IR using a projection
manifest generate ir/recipe.ir.json --projection nextjs --output app/api/

# Compile + generate in one step (recommended after manifest init)
manifest build modules/recipe.manifest
```

The `manifest` CLI ships **inside** `@angriff36/manifest` — there is no separate CLI package to install. Once `@angriff36/manifest` is a dependency, the `manifest` binary is available via `npx manifest` or your package manager's script runner.

<Tip>
  Run `manifest init` once at the start of a project. It creates `manifest.config.yaml` with your framework, auth provider, import paths, and output directory. After that, `manifest build` reads those defaults and you only need to pass flags when overriding them.
</Tip>

## Verifying your install

Run the following to confirm the package resolves correctly and the CLI responds:

```bash
# Check CLI
npx manifest --help

# Smoke-test the compiler from a script
node --input-type=module <<'EOF'
import { compileToIR } from '@angriff36/manifest/ir-compiler';
const { ir, diagnostics } = await compileToIR('entity Ping { property id: string }');
console.log('IR version:', ir?.version ?? 'compiled OK');
EOF
```

## Windows note

On Windows with pnpm, the `manifest` shim has historically exited with code `0` and no output on some setups. If you hit this, invoke the CLI directly:

```bash
pnpm exec manifest --help
# or run the bundled binary directly:
node .\node_modules\@angriff36\manifest\packages\cli\dist\index.js --help
```

If you are on a current release this should not occur; keeping `@angriff36/manifest` up to date is the simplest fix.
