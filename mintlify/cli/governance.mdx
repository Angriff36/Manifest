The governance commands are the static enforcement layer that sits beside the runtime. They check that an application codebase only writes through registered Manifest commands, that emitted routes match the canonical dispatcher, that bypasses are registered, and that the package integrates correctly end-to-end. None of these commands execute a Manifest program — they read the registries, the IR, and your source tree.

<Note>
  These commands were introduced and stabilized across **v0.6.0** and **v0.6.1**. `enforce-surface`
  is the strictest registry-vs-app check; `integration-check` is the recommended pre-deploy umbrella
  for downstream consumers.
</Note>

---

## `manifest enforce-surface`

Detect duplicate or bypass write paths in application code against a registry of canonical Manifest commands. Fails the build when an agent or contributor has invented a direct database write that should have gone through a registered command.

```bash
manifest enforce-surface --root <path> --commands-registry <path> [options]
```

### Required options

<ParamField path="--root" type="string" required>
  Repository or application root to scan.
</ParamField>

<ParamField path="--commands-registry" type="string" required>
  Path to `commands.json` emitted from Manifest IR (see `manifest emit registries`).
</ParamField>

### Optional flags

<ParamField path="--entities-registry" type="string">
  Path to `entities.json` emitted from Manifest IR.
</ParamField>

<ParamField path="--bypass-registry" type="string">
  Path to `bypasses.json` listing approved exceptions.
</ParamField>

<ParamField path="--write-receiver <name>" type="string" default="prisma">
  ORM client identifier that direct-write detectors match on (e.g., `prisma`, `kysely`, `drizzle`). Default is `prisma`.
</ParamField>

<ParamField path="--include" type="glob...">
  Additional include globs (repeatable). Honored as of v0.6.1.
</ParamField>

<ParamField path="--exclude" type="glob...">
  Exclude globs — generated files, build output, fixtures, etc.
</ParamField>

<ParamField path="--strict" type="boolean" default="false">
  Exit non-zero on any error finding.
</ParamField>

<ParamField path="-f, --format" type="string" default="text">
  Output format: `text` or `json`.
</ParamField>

### Example

```bash
manifest enforce-surface \
  --root apps/web \
  --commands-registry manifest-registry/commands.json \
  --bypass-registry manifest-registry/bypasses.json \
  --exclude "**/*.test.ts" "**/generated/**" \
  --strict
```

<Tip>
  As of v0.6.1, `enforce-surface` honors `--include`/`--exclude` correctly and scans `.js` and
  `.jsx` in addition to TypeScript.
</Tip>

## `manifest integration-check`

End-to-end validation that a downstream repo is correctly integrated with the Manifest governance contract. Runs governance audit + bypass audit + dispatcher presence + runtime smoke (audit and outbox adapters) + package shape in one command. Exit code is `0` only if every section passes.

```bash
manifest integration-check [options]
```

### Options

<ParamField path="--root" type="string">
  Downstream repo root (defaults to cwd).
</ParamField>

<ParamField path="--commands-registry" type="string">
  Path to a commands registry JSON.
</ParamField>

<ParamField path="--bypass-registry" type="string">
  Path to a bypass registry JSON.
</ParamField>

<ParamField path="--format" type="string" default="text">
  Output format: `text` or `json`.
</ParamField>

<ParamField path="--strict" type="boolean" default="false">
  Treat warnings as failures.
</ParamField>

<ParamField path="--skip-runtime-smoke" type="boolean" default="false">
  Skip the in-memory `RuntimeEngine` smoke test.
</ParamField>

<ParamField path="--skip-package-shape" type="boolean" default="false">
  Skip the package-shape check.
</ParamField>

<ParamField path="--skip-tarball" type="boolean" default="false">
  Skip the `npm pack --dry-run` sub-step in package-shape.
</ParamField>

<ParamField path="--package-root" type="string">
  Override the `@angriff36/manifest` package root for the package-shape check.
</ParamField>

### Example

```bash
manifest integration-check \
  --root . \
  --commands-registry manifest-registry/commands.json \
  --bypass-registry manifest-registry/bypasses.json \
  --strict
```

## `manifest audit-governance`

Run all five governance detectors in one pass and aggregate findings. This is the umbrella for the individual detectors.

```bash
manifest audit-governance [options]
```

### Detectors

| Detector            | Catches                                                          |
| ------------------- | ---------------------------------------------------------------- |
| `direct-writes`     | Direct database writes that bypass `RuntimeEngine.runCommand`    |
| `event-fabrication` | Manually constructed event payloads that should come from `emit` |
| `route-drift`       | Routes that don't match the canonical dispatcher pattern         |
| `missing-tests`     | Registered commands that have no test coverage                   |
| `bypass-violations` | Bypass entries that no longer match their referenced files       |

### Options

<ParamField path="-r, --root" type="string" default=".">
  Root directory to audit.
</ParamField>

<ParamField path="--only" type="string">
  Comma-separated detector names to run (default: all five).
</ParamField>

<ParamField path="--commands-registry" type="string">
  Path to `commands.json` (enables the `missing-tests` detector).
</ParamField>

<ParamField path="--bypass-registry" type="string">
  Path to `bypasses.json` (enables the `bypass-violations` detector).
</ParamField>

<ParamField path="--write-receiver <name>" type="string" default="prisma">
  ORM client identifier that direct-write detectors match on (e.g., `prisma`, `kysely`, `drizzle`). Default is `prisma`.
</ParamField>

<ParamField path="--strict" type="boolean" default="false">
  Exit non-zero on any error finding.
</ParamField>

<ParamField path="-f, --format" type="string" default="text">
  Output format: `text` or `json`.
</ParamField>

### Example

```bash
# Run all five detectors against the current directory
manifest audit-governance --strict

# Run only the route-drift detector
manifest audit-governance --only route-drift

# Run with both registries enabled
manifest audit-governance \
  --commands-registry manifest-registry/commands.json \
  --bypass-registry manifest-registry/bypasses.json \
  --strict
```

<Note>
  `manifest audit-constitution` is a deprecated alias for `audit-governance`. It still works but
  emits a deprecation warning. Switch to `audit-governance` in new scripts.
</Note>

## `manifest audit-bypasses`

Validate an approved-bypass registry against the canonical schema (`docs/spec/registry/bypasses.schema.json`). Reports missing-file references as errors and expired review dates as warnings (or errors under `--strict-expiry`).

```bash
manifest audit-bypasses --registry <path> [options]
```

### Options

<ParamField path="--registry" type="string">
  Path to bypass registry JSON file.
</ParamField>

<ParamField path="-r, --root" type="string" default=".">
  Root directory used to resolve bypass paths.
</ParamField>

<ParamField path="--strict-expiry" type="boolean" default="false">
  Treat expired `reviewBy` dates as errors instead of warnings.
</ParamField>

<ParamField path="-f, --format" type="string" default="text">
  Output format: `text` or `json`.
</ParamField>

### Example

```bash
manifest audit-bypasses --registry manifest-registry/bypasses.json --strict-expiry
```

## `manifest scan [source]`

Scan `.manifest` source files for configuration issues _before_ runtime. The primary goal: "If scan passes, the code works."

Checks include:

- **Policy coverage** — every command has at least one policy
- **Store consistency** — all `store` targets are recognized

```bash
manifest scan [source] [options]
```

### Options

<ParamField path="-g, --glob" type="string" default="**/*.manifest">
  Glob pattern for manifest files.
</ParamField>

<ParamField path="-f, --format" type="string" default="text">
  Output format: `text` or `json`.
</ParamField>

<ParamField path="--strict" type="boolean" default="false">
  Fail on warnings.
</ParamField>

### Example

```bash
# Scan all .manifest files under modules/
manifest scan modules/ --strict
```

## `manifest doctor`

Run ranked offline diagnostics for source / IR / route drift and duplicate merges. Useful as a single "what's wrong?" command.

```bash
manifest doctor [options]
```

### Options

<ParamField path="--entity" type="string">
  Optional entity to focus the diagnosis on.
</ParamField>

<ParamField path="--command" type="string">
  Optional command to focus the diagnosis on.
</ParamField>

<ParamField path="--route" type="string">
  Optional route path for route-surface correlation.
</ParamField>

<ParamField path="--json" type="boolean" default="false">
  Output JSON instead of text.
</ParamField>

<ParamField path="--src" type="string">
  Source manifest glob pattern.
</ParamField>

<ParamField path="--ir-root" type="path...">
  Compiled IR root directory or directories.
</ParamField>

### Example

```bash
manifest doctor --entity Recipe --command publish
```

## `manifest audit-routes`

Audit generated or hand-written routes for Manifest boundary compliance:

- Writes should execute via `runtime.runCommand`
- Direct reads should include the expected tenant, location, and soft-delete filters

```bash
manifest audit-routes [options]
```

### Options

<ParamField path="-r, --root" type="string" default=".">
  Root directory to audit.
</ParamField>

<ParamField path="-f, --format" type="string" default="text">
  Output format: `text` or `json`.
</ParamField>

<ParamField path="--strict" type="boolean" default="false">
  Fail on warnings and enforce ownership rules as errors.
</ParamField>

<ParamField path="--tenant-field" type="string" default="tenantId">
  Tenant scope field name.
</ParamField>

<ParamField path="--deleted-field" type="string" default="deletedAt">
  Soft-delete field name.
</ParamField>

<ParamField path="--location-field" type="string" default="locationId">
  Location scope field name.
</ParamField>

<ParamField path="--commands-manifest" type="string">
  Path to commands manifest JSON (enables ownership rules).
</ParamField>

<ParamField path="--exemptions" type="string">
  Path to exemptions registry JSON.
</ParamField>

## `manifest emit registries`

Emit machine-readable `commands.json` and `entities.json` registries from a compiled IR. These registries are inputs to `enforce-surface`, `audit-governance`, and `integration-check`.

```bash
manifest emit registries [options]
```

### Options

<ParamField path="--ir" type="string">
  Path to a compiled IR JSON file.
</ParamField>

<ParamField path="--source" type="string">
  Path to a `.manifest` source file to compile and emit from.
</ParamField>

<ParamField path="--out" type="string" default="manifest-registry">
  Output directory for the emitted registries.
</ParamField>

<ParamField path="--no-validate" type="boolean">
  Skip JSON-schema validation of the emitted output.
</ParamField>

<ParamField path="--no-pretty" type="boolean">
  Emit compact JSON (no indentation).
</ParamField>

### Example

```bash
# Emit registries from a compiled IR
manifest emit registries --ir ir/recipe.ir.json --out manifest-registry/

# Or emit straight from source
manifest emit registries --source modules/recipe.manifest --out manifest-registry/
```

## Recommended CI pipeline

A typical pre-deploy check for a downstream consumer:

```bash
# 1. Compile source to IR
manifest build modules/**/*.manifest --code-output app/api/

# 2. Emit registries used by the audit suite
manifest emit registries --ir ir/ --out manifest-registry/

# 3. Run the umbrella check
manifest integration-check \
  --commands-registry manifest-registry/commands.json \
  --bypass-registry manifest-registry/bypasses.json \
  --strict
```

If `integration-check` exits 0, your app is on contract.
