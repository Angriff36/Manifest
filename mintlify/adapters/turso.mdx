The Turso adapter persists entity state in a [Turso](https://turso.tech/) edge database using libSQL. It implements the `Store<T>` interface and stores each entity instance as a JSON blob in a single table.

## Configuration

`TursoStore` is exported from `@angriff36/manifest/stores` (Node.js only). Optional peer: `@libsql/client`.

```typescript
import { TursoStore, generateTursoSchema } from '@angriff36/manifest/stores';

const store = new TursoStore({
  url: process.env.TURSO_DATABASE_URL!,
  authToken: process.env.TURSO_AUTH_TOKEN, // optional for local SQLite
  tableName: 'entities', // default
});
```

### Options

| Option      | Required | Description                                                              |
| ----------- | -------- | ------------------------------------------------------------------------ |
| `url`       | Yes      | Turso / libSQL URL (e.g. `libsql://…` or a local file URL)               |
| `authToken` | No       | Auth token (optional for local SQLite)                                   |
| `tableName` | No       | Single table for JSON blobs (default `entities`)                         |
| `client`    | No       | Pre-built LibSQL client (tests / custom setups)                          |

## Wiring into the runtime

```typescript
import { RuntimeEngine } from '@angriff36/manifest';
import { TursoStore } from '@angriff36/manifest/stores';

const runtime = new RuntimeEngine(
  ir,
  { user: { id: 'user-1' } },
  {
    storeProvider: (entityName) =>
      new TursoStore({
        url: process.env.TURSO_DATABASE_URL!,
        authToken: process.env.TURSO_AUTH_TOKEN,
        tableName: entityName.toLowerCase(),
      }),
  },
);
```

## Edge deployment

- Turso databases are replicated to edge locations for low-latency reads
- Writes go to the primary and propagate to replicas
- Connection pooling is handled by `@libsql/client`

<Note>
  Install the peer: `npm install @libsql/client`. Package pin: `@angriff36/manifest@3.6.41`.
</Note>
