The `realtime` entity modifier marks an entity for live event streaming. When you declare `realtime` inside an entity block, the Next.js projection emits three additional surfaces: an SSE (Server-Sent Events) subscription route, a typed React hook with exponential-backoff reconnect, and a shared singleton runtime accessor that lets command routes and subscriptions observe the same event stream.

## Declaring a realtime entity

Place the `realtime` keyword inside an entity block:

```manifest
entity Task {
  realtime

  property required title: string
  property status: string = "pending"
  property assigneeId: string = ""

  command complete() {
    guard self.assigneeId == user.id or user.role == "admin"
    mutate status = "completed"
    emit TaskCompleted
  }
}
```

- `realtime` is a contextual identifier, not a reserved word — `property realtime: boolean` is still a valid plain property.
- The flag compiles to `realtime: true` on the entity's IR representation.
- **`realtime` has no runtime execution semantics.** It is a projection hint only: identical IR with and without the flag produces identical command results, events, and state.

## Generated surfaces

The Next.js projection exposes three realtime surfaces:

| Surface | Output path | What it emits |
|---------|-------------|---------------|
| `nextjs.subscribe` | `app/api/{entity}/subscribe/route.ts` | SSE `GET` route streaming runtime events for the entity |
| `nextjs.subscriptionHook` | `src/hooks/use{Entity}Subscription.ts` | Typed client-side `EventSource` hook with auto-reconnect |
| `nextjs.sharedRuntime` | `src/lib/manifest-shared-runtime.ts` | Module-scoped `getSharedRuntime()` singleton accessor |

Generate them programmatically:

```typescript
import { NextJsProjection } from '@angriff36/manifest/projections/nextjs';

const projection = new NextJsProjection();

const route = projection.generate(ir, { surface: 'nextjs.subscribe', entity: 'Task' });
const hook = projection.generate(ir, { surface: 'nextjs.subscriptionHook', entity: 'Task' });
const shared = projection.generate(ir, { surface: 'nextjs.sharedRuntime' });
```

Requesting a realtime surface for an entity that is **not** flagged `realtime` produces no artifacts and an `info` diagnostic with code `REALTIME_NOT_ENABLED`. The same applies to `nextjs.sharedRuntime` when no entity in the IR is flagged.

## The SSE subscription route

For each realtime entity, `nextjs.subscribe` generates an endpoint at:

```
GET /api/{entity}/subscribe
```

The route authenticates the request (using your configured auth provider — unauthorized requests are rejected before any stream is opened), obtains the shared runtime, and streams every runtime event whose `subject.entity` matches the entity name. Each SSE message is a `data:` line containing the JSON-serialized runtime event:

```json
{
  "name": "TaskCompleted",
  "channel": "task.completed",
  "payload": { "taskId": "t1" },
  "timestamp": 1765329600000,
  "subject": { "entity": "Task", "command": "complete", "instanceId": "t1" }
}
```

The route unsubscribes from the runtime and closes the stream when the client disconnects (via `request.signal` abort).

## The subscription hook

`nextjs.subscriptionHook` generates a `use{Entity}Subscription` hook — a `"use client"` module that manages the `EventSource` lifecycle:

```typescript
'use client';

import { useTaskSubscription } from '@/hooks/useTaskSubscription';

function TaskFeed() {
  const { connected, lastEvent } = useTaskSubscription({
    onEvent: (event) => {
      console.log(event.name, event.payload);
    },
  });

  if (!connected) return <span className="badge">Reconnecting...</span>;
  return <pre>{JSON.stringify(lastEvent, null, 2)}</pre>;
}
```

### Hook return value

| Field | Type | Description |
|-------|------|-------------|
| `connected` | `boolean` | Whether the SSE connection is currently open |
| `lastEvent` | `{Entity}SubscriptionEvent \| null` | The most recently received event |

### Hook options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `onEvent` | `(event: {Entity}SubscriptionEvent) => void` | — | Callback fired for every event received |
| `initialRetryDelayMs` | `number` | `1000` | Initial reconnect delay after a disconnect |
| `maxRetryDelayMs` | `number` | `30000` | Reconnect delay ceiling |

### Auto-reconnect behavior

When the connection drops, the hook closes the `EventSource` and schedules a reconnect. The delay starts at `initialRetryDelayMs` and **doubles on each failed attempt** up to `maxRetryDelayMs`. A successful reconnect resets the delay. While disconnected, `connected` is `false` and `lastEvent` retains its last value.

## The shared runtime

The event stream is **per-engine-instance and in-memory**. For a subscription to observe events emitted by command execution, the SSE route and the command routes must share one engine. `nextjs.sharedRuntime` generates that singleton:

```typescript
import { getSharedRuntime } from '@/lib/manifest-shared-runtime';

const runtime = await getSharedRuntime();
```

When any entity in the IR is flagged `realtime`, the generated dispatcher and per-command routes (inline execution mode) automatically switch from constructing a per-request runtime to calling `getSharedRuntime()` and setting per-request context via `runtime.replaceContext(...)`.

`externalExecutor` mode is unaffected — the executor owns runtime construction and should call `getSharedRuntime()` itself when realtime is in play.

### Single-instance vs. multi-instance delivery

In-memory, the shared engine's stream is **single-instance**: a subscriber only observes events emitted by commands that executed in its own server process. On a long-lived, single-instance Node server that is all you need.

To fan events across processes (serverless, multiple replicas), configure an `eventBus` in the runtime factory's `RuntimeOptions`. When one is present, the generated `getSharedRuntime()` singleton calls `runtime.connectEventBus()` **once at initialization**, subscribing the process to remote events so events published by commands on other instances reach this stream's local subscribers. Manifest ships two `EventBus` implementations: an in-process `MemoryEventBus` (`@angriff36/manifest/events`, single node / tests) and a `RedisEventBus` (`@angriff36/manifest/events/redis`) for cross-process fan-out.

When **no** bus is configured, the singleton emits a single `console.warn` at init noting that delivery is single-instance only, and continues. For a deployment that advertises multi-instance realtime, set the `realtime.requireEventBus` option to make the generated singleton **throw at init** instead — so a missing bus fails loud at startup rather than silently degrading:

```typescript
const shared = projection.generate(ir, {
  surface: 'nextjs.sharedRuntime',
  options: { realtime: { requireEventBus: true } },
});
```

<Note>
The default companion runtime factory does not inject a bus — supply the `eventBus` through the runtime's `RuntimeOptions` (a custom runtime factory or your app's runtime wiring). See `docs/spec/semantics.md` § "Realtime Entities" and `docs/spec/adapters.md` § "Event Bus".
</Note>

## Runtime `subscribe()` contract

The reference runtime exposes `subscribe(entityName, listener)` — a convenience over `onEvent` that exists regardless of any entity's `realtime` flag:

- The listener receives only events whose `subject.entity === entityName`.
- Events **without** a `subject.entity` are not delivered (use `onEvent` for the unfiltered firehose).
- The return value is an unsubscribe function; after calling it the listener receives no further events.
- Multiple subscribers are independent; listener errors are swallowed and never affect command execution.

```typescript
const unsubscribe = runtime.subscribe('Task', (event) => {
  console.log(event.name, event.subject?.instanceId);
});

// later
unsubscribe();
```

## IR representation

The `realtime` modifier adds a boolean field to the entity IR:

```json
{
  "name": "Task",
  "realtime": true,
  "properties": [...],
  "commands": [...]
}
```

Projections inspect `entity.realtime === true` to decide whether to generate subscription artifacts. Entities without the modifier produce no realtime artifacts.

<Note>
The `realtime` modifier is a declaration only. The SSE transport is implemented by the projection target (the generated Next.js route). The runtime engine itself is transport-agnostic.
</Note>
