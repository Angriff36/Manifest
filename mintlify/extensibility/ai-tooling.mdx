Manifest ships LLM-backed CLI tools for generating fixtures and source from prompts, plus agent-oriented validation. Verified against `packages/cli/src/index.ts` and `@angriff36/manifest@3.6.41`.

## AI-assisted conformance test generator

~~```bash
manifest generate-tests ir/app.ir.json --per-entity 5 --include-negative --format vitest
```~~

> **Correction (2026-07-15) @RYANSIGNED:** `manifest generate-tests` (alias `gen-tests`) is an
> **LLM** fixture generator. Flags are `--feature`, `--category`, `--count`, `--output`,
> `--api-key`, `--model`, `--temperature`, `--max-retries`, `--dry-run`, `--verbose` — not
> `--per-entity` / `--include-negative` / `--format`. Requires `ANTHROPIC_API_KEY`.

```bash
manifest generate-tests src/app.manifest --count 5 --category edge-cases --dry-run
manifest generate-tests --feature "computed properties" --count 3 --output src/manifest/conformance/fixtures
```

| Option | Default | Description |
| ------ | ------- | ----------- |
| `--feature <text>` | — | Focus description for generation |
| `--category <type>` | `edge-cases` | `edge-cases`, `boundary`, `adversarial`, `coverage` |
| `--count <n>` | `3` | Number of fixtures to generate |
| `--output <path>` | conformance fixtures dir | Where to write fixtures |
| `--api-key` / `--model` / `--temperature` | env / defaults | Anthropic settings |
| `--dry-run` | `false` | Preview without writing |
| `--verbose` | `false` | Iteration logs |

Generated fixtures are compiled/validated before write. They are starting points — review them.

## LLM-oriented IR / source validation

~~`manifest validate-ir`~~ / ~~`validateIR` / `repairIR` from `@angriff36/manifest/ir-validator`~~

> **Correction (2026-07-15) @RYANSIGNED — Documentation gap:** There is **no** `validate-ir`
> command and **no** `@angriff36/manifest/ir-validator` export. Use:

```bash
# Schema validation of compiled IR
manifest validate ir/app.ir.json --strict

# Agent-scored diagnostics (text or json)
manifest validate-ai src/app.manifest --format json --min-score 80
```

Automatic “repair IR then continue” is not a shipped public API. Prefer re-emitting source via
`manifest generate-from-prompt` or fixing diagnostics by hand.

## Natural language to Manifest transpiler

~~```bash
manifest generate --from-prompt "…"
```~~

> **Correction (2026-07-15) @RYANSIGNED:** The shipped command is the separate CLI entry
> `manifest generate-from-prompt <prompt>` (not a flag on `manifest generate`). Requires
> `ANTHROPIC_API_KEY`. Options: `--model`, `-o/--output`, `--max-retries`, `--temperature`,
> `--api-key`, `--skip-validation`, `--verbose`. There is no `--format ir` flag.

```bash
manifest generate-from-prompt "An order entity with status tracking and manager approval over 1000" -o generated.manifest
manifest generate-from-prompt "A task tracker with assignees" --model claude-sonnet-4-20250514 --verbose
```

Always compile and validate the result:

```bash
manifest compile generated.manifest --diagnostics
manifest validate-ai generated.manifest --min-score 80
```

<Note>
  LLM output is an emitter, not a validator. Review guards, policies, and domain completeness
  before shipping. Package pin: `@angriff36/manifest@3.6.41`.
</Note>
